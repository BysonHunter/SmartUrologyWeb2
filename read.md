# SmartUrologyWeb2

Веб-приложение для анализа DICOM-исследований (урология) с автоматическим поиском объектов (почки/камни), расчетом параметров и формированием PDF-отчета.

## Что делает проект

- Принимает ZIP-архив с DICOM-файлами через веб-интерфейс.
- Находит доступные серии исследования и дает выбрать нужную.
- Запускает пайплайн обработки:
  - чтение DICOM серии;
  - детектирование объектов;
  - расчет параметров найденных объектов;
  - сбор результатов и подготовка PDF-отчета.
- Показывает прогресс обработки и отдает готовый PDF для скачивания.

## Основные компоненты

- `main_frontend.py` — Flask-приложение, маршруты, сессии, прогресс, загрузка/выдача результатов.
- `main_backend.py` — основной сценарий обработки DICOM-данных.
- `readDicom/` — чтение DICOM и вспомогательные функции.
- `detectObj/` — детектирование объектов и веса моделей (`detectObj/weights`).
- `buildObj/` — расчет параметров и формирование результирующих данных.
- `templates/`, `static/` — HTML-шаблоны и фронтенд-ресурсы.

## Стек

- Python 3.11
- Flask
- pydicom
- OpenCV
- PyTorch / Ultralytics
- NumPy, SciPy, pandas, matplotlib и др.

## Локальный запуск

1. Создать и активировать виртуальное окружение.
2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Запустить приложение:

```bash
python main_frontend.py
```

4. Открыть в браузере:

`http://localhost:5000`

## Запуск через Docker

Сборка и запуск:

```bash
docker compose up --build
```

После запуска сервис доступен на `http://localhost:5000`.

## Папки с данными

- `in/` — входные данные (загруженные и распакованные DICOM).
- `workdir/` — промежуточные файлы обработки.
- `out/` — итоговые результаты и PDF-отчеты.

## Особенности

- Для каждого пользователя создается отдельная сессия и отдельные директории обработки.
- Есть фоновая очистка старых сессий и временных данных.
- Поддерживается асинхронная обработка (в отдельном потоке) с отслеживанием статуса.
